1. Что такое виртуальный хост в Nginx?
Исторически термин «виртуальный хост» (Virtual Host) пришел из веб-сервера Apache. В терминологии Nginx это называется серверным блоком (Server Block).
По сути, это механизм конфигурации, который позволяет одному физическому серверу (или виртуальной машине) с одним IP-адресом обслуживать сразу несколько совершенно разных веб-сайтов (доменов). Для внешнего пользователя это выглядит так, будто каждый сайт работает на своем собственном независимом сервере.
2. Зачем он нужен?
Использование серверных блоков решает сразу несколько важных задач:
 Экономия ресурсов и денег: Тебе не нужно арендовать отдельный сервер и покупать отдельный выделенный IP-адрес для каждого нового сайта. Десятки или сотни сайтов могут крутиться на одной недорогой машине.
 Изоляция настроек: Для каждого сайта можно задать свои уникальные правила: отдельную папку с файлами сайта (⁠root⁠), свои файлы для записи логов доступа и ошибок, разные версии PHP или правила переадресации.
 Удобство администрирования: При добавлении нового проекта достаточно просто создать новый текстовый файл с конфигурацией блока ⁠server⁠, не трогая настройки других работающих сайтов.
3. Как один сервер обслуживает несколько сайтов (механизм работы)
Вся "магия" строится на протоколе HTTP. Когда браузер пользователя обращается к серверу, он не просто стучится на IP-адрес, он обязательно передает специальный HTTP-заголовок ⁠Host⁠, в котором указано имя домена (например, ⁠Host: site-a.com⁠).
Алгоритм работы Nginx:
1. Nginx принимает входящее соединение на 80 (HTTP) или 443 (HTTPS) порт.
2. Читает HTTP-запрос и смотрит на заголовок ⁠Host⁠.
3. Ищет в своих конфигурационных файлах блок ⁠server⁠, у которого директива ⁠server_name⁠ совпадает с запрошенным доменом.
4. Отдает файлы из той директории, которая указана внутри найденного блока.
